¿Que es mongodb?

MongoDB es un sistema de gestión de bases de datos (DBMS) no relacional de código abierto que utiliza documentos flexibles en lugar de tablas y filas para procesar y almacenar diversas formas de datos. Como solución de base de datos NoSQL, MongoDB no requiere un sistema de gestión de bases de datos relacionales (RDBMS), por lo que proporciona un modelo de almacenamiento de datos flexible que permite a los usuarios almacenar y consultar tipos de datos multivariados con facilidad. Esto no únicamente simplifica la gestión de bases de datos para los desarrolladores, sino que también crea un entorno altamente escalable para aplicaciones y servicios multiplataforma. Los documentos o colecciones de documentos de MongoDB son las unidades básicas de datos. Con un formato JSON binario (notación de objetos de secuencia de comandos de Java), estos documentos pueden almacenar varios tipos de datos y distribuirse en diferentes sistemas. Dado que MongoDB está diseñado bajo un esquema dinámico, los usuarios tienen una flexibilidad de, al crear registros de datos, consultar colecciones de documentos a través de la agregación de MongoDB y analizar grandes cantidades de información.

Caracteristicas

  • NOSQL

    MongoDB es un sistema de gestión de bases de datos NoSQL, lo que significa que no utiliza un esquema de bases de datos relacional tradicional. En su lugar, utiliza un modelo de documentos para almacenar y recuperar datos.

  • Document-based

    En MongoDB, los datos se almacenan en documentos JSON (JavaScript Object Notation) o BSON (Binary Serialized Object Notation). Esto permite una gran flexibilidad en la estructura de los datos y facilita la manipulación de los mismos.

  • Schema-less

    MongoDB no requiere un esquema previo para crear una base de datos. Esto significa que puedes agregar campos y estructuras a tus documentos en cualquier momento, sin necesidad de cambiar la estructura de la base de datos.

  • Scalability

    MongoDB es escalable, lo que significa que puedes agregar más nodos a tu cluster para manejar un aumento en la carga y la cantidad de datos.

  • High availability

    MongoDB ofrece alta disponibilidad, lo que significa que puedes configurar tu cluster para que esté disponible en caso de fallos o mantenimiento.

  • Replication

    MongoDB ofrece replicación, lo que significa que puedes crear copias de seguridad de tus datos en diferentes nodos para garantizar la disponibilidad y la recuperación en caso de fallos.

Funcionamiento

MongoDB almacena objetos de datos en colecciones y documentos en lugar de las tablas y filas que se utilizan en las bases de datos relacionales tradicionales. Las colecciones comprenden conjuntos de documentos, que son equivalentes a tablas en una base de datos relacional. Los documentos consisten en pares clave-valor, que son la unidad básica de datos en MongoDB. La estructura de un documento se puede cambiar simplemente añadiendo campos nuevos o eliminando los existentes. Los documentos pueden definir una clave principal como identificador único y los valores pueden ser una variedad de tipos de datos, incluidos otros documentos, matrices y matrices de documentos.

Desventajas

  • No tiene Joins para consultas

    Esta es una de las grandes desventajas de MongoDB, y es que no permite hacer Joins para consultas, es decir, consultas en las que se combinan o relacionan diferentes tablas. La forma de ejecutar este tipo de consultas en MongoDB se hace de otra manera.

  • Falta de relaciones estrictas

    A diferencia de las bases de datos relacionales, MongoDB no impone relaciones estrictas entre documentos, lo que puede dificultar la representación de relaciones complejas entre entidades. Esto puede requerir la implementación de soluciones adicionales para manejar relaciones entre documentos.

  • Ecosistema joven

    En comparación con bases de datos relacionales como MySQL o PostgreSQL, MongoDB tiene un ecosistema de herramientas, bibliotecas y recursos de terceros menos maduro. Esto puede dificultar encontrar soluciones a problemas específicos o integrar la base de datos con otras aplicaciones.
